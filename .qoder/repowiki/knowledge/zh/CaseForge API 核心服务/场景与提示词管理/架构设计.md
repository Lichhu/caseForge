- 采用 NestJS 标准分层架构：`controller` 暴露 REST 接口，`service` 封装业务逻辑，`entity` 定义 TypeORM 数据模型，`dto` 负责请求校验。
- 通过 `index.ts` 中的 `ScenarioModule` 聚合依赖，利用 `TypeOrmModule.forFeature` 注册 `ScenarioEntity` 和 `PromptEntity`。
- 业务层实现多租户/用户隔离逻辑，通过 `user-scope` 工具函数进行数据访问控制（如 `scopedWhere`, `assertOwned`）。
- 支持场景与提示词的级联操作，在更新场景时可选择性全量替换关联的提示词列表。
