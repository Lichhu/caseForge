- **分层架构**：采用标准的 Controller-Service-Entity 三层结构。`ApiTestController` 统一暴露 RESTful 接口，7 个 Service（如 `ApiDocService`, `ApiExecutionService`）处理核心业务逻辑，10 个 TypeORM Entity 映射数据模型。
- **核心流程**：支持从 Excel/文本上传并结构化 API 文档 (`api-doc.service.ts`)，基于文档端点生成测试用例，配置多环境变量与密钥，通过执行集 (`execution-set`) 组织用例，并利用并发引擎 (`api-execution.service.ts`) 发起 HTTP 请求进行断言验证。
- **工具支撑**：内置变量替换 (`variable-substitute.util.ts`)、断言执行器 (`assertion-runner.util.ts`) 及 AI 辅助生成工具，实现动态参数注入与多维度结果校验。
- **依赖集成**：通过 `MinioStorageModule` 存储原始文档，利用 `AiWorkflowModule` 增强用例生成，并通过 `TypeOrmModule` 管理持久化状态。
