- **分层架构**：遵循 Controller-Service-Entity 模式，`StructDocController` 处理 HTTP 请求，`StructDocService` 编排业务逻辑，TypeORM 实体映射数据持久化。
- **异步任务队列**：通过 `StructRequirementQueueService` 实现基于数据库的持久化任务队列（`struct_requirement_job`），支持服务重启后的任务恢复与状态对账。
- **并发控制**：利用 `structuring-concurrency.ts` 中的内存信号量限制全局 AI 结构化并发数，防止资源过载。
- **外部集成**：依赖 `MinioStorageModule` 存储原始文档与结构化结果，通过 `AiWorkflowModule` 调用 AI 工作流进行文档解析与总结生成。
- **解析引擎**：内置 `struct-doc.parser.ts` 提供多策略 Markdown 解析能力，从 AI 输出中提取系统、功能模块及测试要点。
