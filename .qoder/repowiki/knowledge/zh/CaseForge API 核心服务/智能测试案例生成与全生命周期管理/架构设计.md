- **分层架构**：采用 NestJS 模块化设计，通过 `CaseEditorController` 暴露 REST API，核心逻辑下沉至 `service` 层。
- **核心服务职责**：`CaseWorkspaceService` 负责业务编排（需求解析、AI 调用协调）；`CaseEditorService` 处理案例树（Tree Structure）的持久化与增量更新（Diff-based）；`CaseGenerateQueueService` 实现基于 DB 的异步任务队列与公平调度。
- **数据持久化**：使用 TypeORM 管理 `CaseTreeEntity`（节点结构）与 `CaseNodeMetadataEntity`（节点属性），支持大规模树形数据的批量插入与差异更新（见 `case-tree-diff.util.ts`）。
- **外部集成**：通过 `AiWorkflowModule` 对接 LLM 进行案例生成，通过 `MinioStorageModule` 存储 Skill 模板，通过 `TestPlatformModule` 同步至外部测管平台。
- **导出与同步**：`ExportService` 支持 Excel/XMind 格式导出，`CaseTestPlatformSyncService` 负责将内部案例模型映射并同步至第三方测试管理系统。
