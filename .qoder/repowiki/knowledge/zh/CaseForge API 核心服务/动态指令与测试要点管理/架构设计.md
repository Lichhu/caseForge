- 采用标准 NestJS 分层架构：`controller` 处理 HTTP 请求，`service` 封装业务逻辑，`entity` 定义 TypeORM 数据模型。
- 核心数据模型包括 `TestPointInstructEntity`（一对一存储指令状态与自然语言）和 `TestPointPromptEntity`（多对多关联场景提示词）。
- 模块通过 `TypeOrmModule.forFeature` 注入相关实体仓库，并导出 `DynamicInstructService` 供其他模块复用。
- 列表查询采用“先查 ID 排序再查详情”的两阶段加载策略，并结合自定义 SQL 片段实现基于业务状态的优先级排序。
