- **状态管理层**：基于 Pinia 的 `caseForge` Store (`apps/web/src/stores/caseForge.ts`) 作为核心，管理项目生命周期、三阶段工作流（文档/指令/编辑台）状态、异步任务轮询（结构化/AI生成）及本地缓存同步。
- **视图编排层**：`CaseForgeDashboardView.vue` 负责顶层布局与阶段切换，集成 `useImmersiveWorkspace` Composable 实现全屏沉浸式交互与悬浮 Orb 导航。
- **功能组件层**：
  - **编辑台**：`CaseTreeWorkbench.vue` 集成 `mind-elixir` 实现思维导图视图，并提供 `CaseTreeExcel.vue` 实现类 Excel 的行列式编辑，支持层级折叠、单元格合并及批量操作。
  - **辅助模态框**：`CaseAddModal.vue` 和 `CaseSelectionModal.vue` 处理用例的新增归属选择与批量同步/导出筛选。
  - **基础节点**：`TreeNode.vue` 提供递归树形结构的渲染与基础增删改交互。
- **数据流向**：组件通过 Store Actions 触发 API 请求，Store 更新响应式状态后驱动视图重绘；本地编辑通过 `dirty` 标记与手动保存机制同步至后端。
