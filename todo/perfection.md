# 案例生成平台
## 动态指令
- 场景维护
    - 用户可以查询 system和自己维护的，场景提示词包。
    - 生成方式默认为全量覆盖

## 案例编辑台
- 案例需要存储正向、反向，案例性质字段。
- 案例需要存储高、中、低，优先级字段。
- Excel支持查询，先筛选功能点，再模糊查询案例，查询框放在(收起\展开)后面。
- 同步测管 测管平台需要对接。
- 失焦自动保存，需要保存提示。
- 导出去掉json，只需要留 xmind和excel。
- 同步测管默认全部不选，需要展示案例就行。
- 导出Excel 需要按照默认格式，导出和同步测管一样弹窗选择案例导出。

# 接口测试平台
## 项目管理
- 手动新建项目，XQxxxx-xxxx-xx 是必须要的，需求为维度。
- 每个需求下对应有一个交易码列表（每条交易码对应一个接口）。
- 点击交易码 跳到该交易码下的 四步流程页面。
    - 四步流程页面
        - 接口文档（已识别接口 这个列表不需要，可以预览和编辑 结构化的接口文档）
            上传EXECL文档，解析文本，解析的文本 加 skill，生成结构化接口文档。
        - 案例生成
            接口的案例列表展现，与动态指令页面结构差不多（左边展示案例card， 右边展示当前案例的各种字段数据）。
            案例针对当前交易码，或者针对当前接口，通过ai生成的。
            案例的生成，是从各个维度测试当前接口（skill 加上 结构化接口文档 返回json结构的案例数组）。
            支持手动添加案例。
            案例需要 案例名称，案例编号，案例描述，案例类型，备注，状态，请求报文，预期结果，创建人，负责人，交易码。
        - 执行平台
            执行平台需要有一个环境维护，点击弹出维护弹窗。（环境维度 -> 环境服务维度）。
            执行集平台，每个执行集内不可以存在重复案例，一个案例可被多个执行集引入。
            执行集展示执行状态，直接后的结果。            
        - 结果报表
            展示执行集执行结果，案例执行结果，案例执行详情。
