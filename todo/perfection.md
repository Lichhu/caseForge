# 接口测试平台
## 项目管理
    - 四步流程页面
        - 接口文档（已识别接口 这个列表不需要，可以预览和编辑 结构化的接口文档）
            上传EXECL文档，解析文本，解析的文本 加 skill，生成结构化接口文档。
        - 案例生成
            接口的案例列表展现，与动态指令页面结构差不多（左边展示案例card， 右边展示当前案例的各种字段数据）。
            案例针对当前交易码，或者针对当前接口，通过ai生成的。
            案例的生成，是从各个维度测试当前接口（skill 加上 结构化接口文档 返回json结构的案例数组）。
            支持手动添加案例。
            案例需要 案例名称，案例编号，案例描述，案例类型，备注，状态，请求报文，预期结果，创建人，负责人，交易码。
        - 执行平台
            执行平台需要有一个环境维护，点击弹出维护弹窗。（环境维度 -> 环境服务维度）。
            执行集平台，每个执行集内不可以存在重复案例，一个案例可被多个执行集引入。
            执行集展示执行状态，直接后的结果。            
        - 结果报表
            展示执行集执行结果，案例执行结果，案例执行详情。
